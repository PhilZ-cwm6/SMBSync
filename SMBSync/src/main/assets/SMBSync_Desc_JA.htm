1.    機能

SMBSyncはAndroid端末とPCやNASとの間でSMB/SAMBAを使用してファイルの同期を行うためのツールです。同期はマスターからターゲットへの一方向で、ミラー、移動、コピーの３種類が使用できます。(Android端末の内部ストレージとSDCARD間での同期も可能です＊１)

SMBSyncのスケジュール機能または外部アプリケーション（TaskerやAutoMagicなど）により定期的に同期を開始する事が可能です。外部アプリケーションとのインターフェースについては他のアプリケーションからの起動を参照してください。

 

        ミラー

マスター側のディレクトリーとファイルをターゲット側に差分コピーし、コピー終了後にマスター側に存在しないターゲット側のファイルとディレクトリーを削除する。

        移動

マスター側のディレクトリーとファイルをターゲット側にコピーし、ターゲット側にコピーしたマスター側のファイルを削除する。（ただし、マスターとターゲットに同名でファイルサイズと更新日時が同じファイルはコピーせずマスター側のファイルを削除）

        コピー

マスター側のディレクトリーに含まれるファイルをターゲット側に差分コピーする。（ただし、マスターとターゲットに同名でファイルサイズと更新日時が同じ場合はコピーしない）

下記の３条件のうちいずれかが成立した場合に差分ファイルと判定。

        ファイルが存在しない

        ファイルサイズが違う

        最終更新日時が3秒以上違う(秒数はオプションにより変更可能)

 

＊１:Android 4.4ではアプリ固有ディレクトリーのみ同期可能。Android 4.4以前と5.0以降では全てのディレクトリーで同期が可能。

2.    使用方法

ミラーと移動はファイルが削除されます、コピーはファイルを上書きしますので、テストディレクトリーを作成しテストしてからご使用ください。

同期を行うにはローカル、リモート、同期のプロファイルを作成することが必要です。ローカルからローカルへの同期（内部ストレージからUSBストレージなど）ではリモートプロファイルは不要です。

        ローカルプロファイルはローカル側（Android側）のディレクトリーを指定します。

        リモートプロファイルはリモート側（NASやPCなどのSMBサーバー）のディレクトリーを指定します。

        同期プロファイルはローカルまたはリモートのプロファイルをマスターまたはターゲットとして指定し、同期タイプを指定します。

同期は同期プロファイル毎に行います。同期する順序はプロファイルタブに表示されている順番です。（同期プロファイルの名前の昇順になります）

プロファイルの作成例

 

上記の例ではWi-Fiで接続されたAndroid端末にPCとNASのフォルダーからファイルを同期するプロファイルを示している。

        SYNC1はPCのフォルダーからLocalの/mnt/sdcard/DCIM/DIR-PCにjpgファイルをミラーで同期をおこないます。

        SYNC2はNASのフォルダーからLocalの/mnt/sdcard/DCIM/DIR-NASにjpgファイルをミラーで同期をおこないます。

        ディレクトリー名

ローカルのディレクトリーを指定する

プロファイル名はローカル、リモート、同期で重複できません。

        ユーザーID/パスワードを使用する

ユーザーID/パスワードを使用するかどうかを指定する

        ユーザー名

        パスワード

（一覧ボタンを押すと選択可能な共有名が表示されますので、共有名をクリックし、了解を押す）

        共有名

共有名を指定する

        一覧（ディレクトリー）

リモートのディレクトリー一覧を表示する

（一覧ボタンを押すと選択可能なディレクトリーが表示されますので、ディレクトリーをクリックし、了解を押す）

リモートのディレクトリーを指定する

2.2.3 同期プロファイルの作成

コンテキストメニューから同期プロファイルの追加をクリック。

 

＊１：ボタンをタップするとプロファイルを編集できます。

        プロファイルを有効にする

チェックされているとプロファイルを有効にする

        プロファイル名

プロファイルの名前を指定する

        マスタープロファイル

同期の元となるプロファイルを指定する。左端のボタンを押すことにより選択されたプロファイルを編集できます。

        ターゲットプロファイル

同期先となるプロファイルを指定する。左端のボタンを押すことにより選択されたプロファイルを編集できます。

マスターとターゲットプロファイルはリモートとローカルまたはローカルとローカルの組み合わせにすることが必要です。

        同期方式

ミラー、コピーと移動の方式を選択

        フィルター（ファイル）

ファイル選択フィルターを参照

        フィルター（ディレクトリー）

ディレクトリー選択フィルターを参照

        マスターディレクトリーのファイルとディレクトリーを処理

チェックした場合はマスタープロファイルで指定されたディレクトリーに存在するファイルとディレクトリーを処理します。（省略時値はチェック有りです）

チェックしていない場合はマスタープロファイルで指定されたディレクトリーに存在するディレクトリーだけを処理します。

        サブディレクトリーを同期する

チェックするとサブディレクトリーを同期する。

        上書コピーまたは削除の前に確認する

チェックした場合はファイルの削除と上書きするときに確認ダイアログを表示します。

        リモートファイルの最終更新時刻をローカルファイルに合わせない

通常は使用しない。

リモートの制限によりSmbFile#setLastModified()が失敗する場合に有効にしてください。これにより、ローカルとリモートのファイルの差分判定はファイルの存在の有無とファイルサイズで判定されます。

        強制的にSMBSyncで最終更新日時を取得

通常は使用しない。

Java File#setLastModified()を使用しないで、SMBSync独自方式でローカルファイルの最終更新日時を保守する。

        同期中にネットワークエラーが発生したらリトライする

リモート側のエラーに限り同期をリトライする。リトライは最大3回行う、それぞれのリトライはエラー発生から30秒経過後に行う。

        空のディレクトリーを同期する

チェックすると空のディレクトリーを同期する。（ターゲットに空のディレクトリーを作成する）

        隠しディレクトリーを同期する

チェックすると隠しディレクトリーを同期する。（Androidでは隠しディレクトリーの名前は「．」から始まりますが、WindowsやSambaではディレクトリーの名前ではなくディレクトリーのビットであらわされますので、作成されたディレクトリーは隠しディレクトリーになりません）

        隠しファイルを同期する

チェックすると隠しファイルを同期する。（Androidでは隠しファイルの名前は「．」から始まりますが、WindowsやSambaではファイルの名前ではなくディレクトリーのビットであらわされますので、作成されたファイルは隠しファイルになりません）

2.4 ファイル選択フィルター

        選択/除外

選択フィルターまたは除外フィルターの指定

        追加

        フィルター

フィルター値です。DSC*.JPGなどと入力してください。

ファイルフィルターの指定例

 

選択フィルターとして「FILE*.*」、除外フィルターとして「FILE*.JPG」の二つを指定した場合に処理するファイルはFIILE01.TXT、FIILE02.TXTとFIIL-2.mpgになります。

3.    タブの説明

3.1.1 基本表示モード

プロファイル名をタップするとプロファイルの編集ができます。また、下部のボタンを押すことで操作ができます。

 全ての有効な同期プロファイルを同期します

 プロファイル作成ウィザードを起動します

 ローカルプロファイルの追加

 リモートプロファイルの追加

 同期プロファイルの追加

 全てのプロファイルを選択

 

 

3.1.2 選択モード

全選択アイコンまたは、プロファイル名を長押しすることにより選択モードに画面表示が切り替わります。基本表示モードするには画面左上の完了ボタン(左矢印)または戻るキーを押します。

右端のチェックボックスをタップすると選択・非選択を切り替えられます。またプロファイル名をタップしても切り替えできます。

 選択された有効な同期プロファイルを同期します

 選択されたプロファイルを有効にする

 選択されたプロファイルを無効にする

 選択されたプロファイルのコピー（複数選択では無効）

 選択されたプロファイルの改名（複数選択では無効）

 プロファイルの削除

 全てのプロファイルを選択

 全てのプロファイルの選択解除

       毎時（毎時xx分に実行など）

       毎日（毎日xx時xx分に実行など）

       曜日指定（月曜日と金曜日のxx時xx分に実行など）

        SMBSyncをバックグラウンド実行

同期開始後にホームスクリーンを表示する

        全ての有効なプロファイルを同期する

チェック無しにすると同期リストを選択することができます

        同期リストの編集

ボタンを押すと編集画面が表示されます

        同期開始時にWi-Fiをオンにする

        遅延時間

Wi-Fiをオンにした後に指定した時間が経過したら同期を開始する

        同期終了後にWi-Fiをオフにする

4.8設定

4.8.1 同期

        無線LANオプション

無線LANの状態で同期を実行する条件を指定します。

       無線LANがオフでも同期を実行可能にする

       無線LANがオンでアクセスポイントに接続していなくても同期を実行可能にする

       無線LANがオンでアクセスポイントに接続しているときに同期を実行可能にする

        自動同期起動

チェックされている場合は、アプリケーション起動後に有効なプロファイル全ての同期を実行します。

        自動停止（自動同期起動が選択されている場合に有効）

チェックされている場合は、同期が終了したらアプリケーションを終了します。

        バックグラウンド実行（自動同期起動が選択されている場合に有効）

チェックされている場合は、ホームスクリーン(バックグランド実行)を表示します。

        エラー時のオプション

チェックされている場合は、同期実行時にエラーが発生しても無視します。

        画面消灯

同期実行中の画面消灯オプションを指定する（指定の有無にかかわらず電源ボタンを押すことで画面は消灯します）

       同期中に画面消灯を許可する

       同期中は画面消灯を許可しない

画面消灯時に同期が開始された時に画面が点灯します

       画面が画面消灯されていない時に同期を開始した場合は画面消灯を許可しない

        Wi-Fiロック

チェックされている場合は、同期実行中にWi-Fiロックを取得する

        リモートコピー先ファイルの保存

チェックされている場合は、リモートへのファイルコピー時にコピー完了までコピー先のファイルを保存する(コピーが中止された場合はファイルの内容は変更されません)

        ローカルでのコピー先ファイルの保存

チェックされている場合は、ローカルへのファイルコピー時にコピー完了までコピー先のファイルを保存する(コピーが中止された場合はファイルの内容は変更されません)

4.8.2 ユーザーインターフェース

        代替UI

「他の項目が選択されていないときにクリックしたら編集、他の項目が選択されているときは項目を選択」を有効にする

        警告メッセージ抑止

マウントポイント混在の警告メッセージを抑止するかどうかを決めます。

        デバッグメッセージの表示

デバッグメッセージをメッセージタブに表示するかどうかを決めます。

        バックグラウンド終了の通知

バックグラウンド実行状態でアプリケーションが終了した場合（自動終了が有効な場合）のNotificationでの通知方法を選択します。

バックグラウンドで実行中は必ずNotificationに通知され、状態が確認できます。

        同期終了時の通知音の再生

同期終了時に省略時通知音を再生するかどうかを設定します。

        同期終了時のバイブレーション

同期終了時に振動させるかどうかを設定します。

        画面テーマ

画面テーマにライトを使用するかどうかを設定します。

4.8.3 ログ

        ログ

4.8.4 MediaScannerオプション

        非メディアファイル

非メディアファイルをコピー終了時にMediaScannerでスキャンするかどうかを指定する

        外部ストレージ

USBやSDCARD等の外部メディアへのコピー終了時にMediaScannerでスキャンするかどうかを指定する

4.8.5 差分ファイル判定

        最終更新時刻の差

差分ファイルとするリモートとローカル ファイルの最終更新時刻の差を秒数で指定します。（初期値は３秒）

        メディアストアファイルの最終更新時刻

メディアストア（DCIMやMusicフォルダー等）のファイルについて、ファイル最終更新時刻の取り扱いを選択します。

       最終更新時刻だけで差分を判定

       最終更新時刻と最終更新時刻＋タイムゾーンで差分を判定

       ファイル最終更新時刻を使用しない

端末再起動によるメディアストアフォルダーのファイルの最終更新時間が書き換わりファイルが毎回コピーされる場合は（２）を試してください。

JCIFS拡張セキュリティ（初期値：チェック無し）

        ＳＭＢパフォーマンス

パフォーマンスクラスを選択。 (低、中、大 など)

詳細を指定するためにはSMB tuning parametersを選択してください。

通常は使用しない。指定した後に初期値にする場合は値を削除して空白にしてください。

5.1.4 SyncProfile

AutoStartが指定された場合、同期するプロファイルを指定することができます。

存在しないプロファイルや無効なプロファイルを指定しプロファイルが選択されていない状態になった時は、有効なプロファイル全てが同期されます。

タイプ：String[]

値:有効な同期プロファイル

 

目次に戻る

 

5.1.5 StartupParms

上記の４つのオプションをひとつで指定する。それぞれのオプションはカンマで区切り指定する。

タイプ：String

値:”true/false,true/false,true/false,SyncProf”

アイテム１：AutoStart=trueまたはfalse

アイテム２：AutoTerm=trueまたはfalse

アイテム３：Background=trueまたはfalse

アイテム４：SyncProfileのプロファイル名をカンマで区切り指定する

例：StartupParms=”true,false,true,SPROF1,S-PROF2”

(AutoStartが有効、AutoTermは無効、バックグラウンド実行は有効、同期するプロファイルはSPROF1とS-PROF2)

5.2 サンプルコード

下記のコードはActivityから起動するサンプルです。

 

public class MainActivity extends Activity {

@Override

protected void onCreate(Bundle savedInstanceState) {

super.onCreate(savedInstanceState);

setContentView(R.layout.activity_main);

}

@Override

protected void onResume() {

super.onResume();

 

PackageManager pm = getPackageManager();

Intent in=pm.getLaunchIntentForPackage("com.sentaroh.android.SMBSync");

in.setAction(Intent.ACTION_MAIN);

in.setFlags(Intent.FLAG_ACTIVITY_NEW_TASK);

in.putExtra("AutoStart", true);

in.putExtra("AutoTerm", true);

in.putExtra("Background", true);

String[] prof=new String[] {"S-PGMS","S-Pictures"};

in.putExtra("SyncProfile", prof);

startActivity(in);

 

}

}

6.    サンプルプロファイルについて

下記の3種類の同期についてプロファイルを提供している。

         リモートからローカルにミラー(S- DOWNLOAD-MY-PICTURE)

         ローカルからローカルにミラー(S-BACKUP-TO-USB-MEMORY)

         ローカルからリモートにミラー(S-BACKUP-MY-PICTURE)

6.1 ネットワーク

Wi-Fiでリモート（PCまたはNASなどの）SMB/Sambaによる共有ができることを前提としている。リモートサンプルプロファイルではリモートIPアドレスを192.168.0.2に想定しているので、使用者の環境に合わせてIPアドレスを変更してください。

 

6.2 リモート側の設定

リモート側のファイル共有設定を下記に想定しているので、使用者の環境に合わせてリモートプロファイル（R-SAMP-DOWNLOADとR-SAMP-UPLOAD）を変更してください。

 

ユーザーID       ：TESTUSER

パスワード        ：PSWD

共有名                ：SHARE

ディレクトリー ：\Android\Pictureおよび\Android\DCIM（\Android\Pictureは作成されている必要があります、\Android\DCIMはサンプルで作成されます）
